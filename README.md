# Desk Manager

一个 Windows 桌面文件和待办事项管理工具。

## 功能特性

- 🔒 **边缘吸附**: 靠近桌面边缘自动吸附并收缩（类似QQ侧边栏）
- 📁 **文件管理**: 卡片式管理桌面文件和文件夹
- ✅ **待办事项**: TodoList功能，支持主题、时间、描述
- 🔔 **系统通知**: 待办任务快到时间时弹出Windows通知
- 🎨 **透明度调节**: 可自定义窗口透明度
- 🖼️ **美观设计**: 圆润的UI设计，透明背景

## 技术栈

- **Electron**: 桌面应用框架
- **React**: UI框架
- **TypeScript**: 类型安全
- **Vite**: 快速构建工具

## 使用说明

1. **文件管理**: 拖拽文件或文件夹到卡片区域进行管理
2. **待办事项**: 点击"+"按钮添加待办任务
3. **锁定窗口**: 点击🔒按钮固定窗口位置
4. **调节透明度**: 点击⚙️按钮打开设置面板

 ## 需求
   1. 窗口锁定/解锁状态

  - 锁定状态：窗口不能拖动，不能改变大小
  - 解锁状态：窗口可以拖动，可以通过边框拖动改变大小（Windows风格）

  2. Windows风格视觉反馈

  - 当光标靠近窗口边框时，显示不同的光标样式（↔️、↕️、↖️ 等）
  - 窗口边框应该有高亮效果

  3. 自动隐藏功能（QQ风格） - 类似QQ在任务栏边缘的行为

  - 启用时：
    - 当鼠标离开窗口 1 秒后，窗口自动收缩到屏幕右边，只显示 5px 的边框
    - 当鼠标进入屏幕右侧 50px 的触发区域时，窗口自动弹出显示
    - 当鼠标再次离开窗口，1 秒后再次收缩
  - 禁用时：窗口保持正常显示，不收缩

  4. 窗口边界约束

  - 窗口不能拖动超出屏幕边界
  - 窗口不能被拖动到屏幕外

  5. UI 控制

  - 有一个锁定按钮来切换 锁定/解锁 状态
  - 有一个自动隐藏开关来启用/禁用 QQ 风格收缩功能
  - 设置面板可以控制窗口透明度和这两个功能

## 待解决bug
 窗口拖动Bug详细描述

  问题现象：
  在解锁状态下，当你拖动窗口往右侧靠近屏幕右边框时：
  1. 窗口正常拖动，当接近右边界时被阻止继续向右移动（这是正确的边界约束）                                                                                   
  2. 但是，当鼠标继续往右边框方向移动时，窗口会往左侧变宽 ❌

  换句话说：窗口的右侧边框保持不动（因为已经贴着屏幕右边），但窗口的宽度却在增加，导致窗口向左侧扩展。

  触发条件：
  - 窗口必须处于解锁状态
  - 窗口贴住屏幕右边框
  - 继续拖动鼠标向右（虽然窗口不能再往右移了）

  我之前的分析：
  这个bug是由于在 move 事件处理中调用 setPosition() 来约束边界时：
  1. setPosition() 调用会触发新的 move 事件
  2. 新的 move 事件再次获取窗口的 getBounds()，此时宽度可能已经被Electron的原生行为修改
  3. 导致使用了错误的宽度值进行边界计算
